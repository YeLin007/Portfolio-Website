Pokémon Card App

React Native CRUD State Management

The Pokémon Card App is a small mobile project where users can browse a list of Pokémon cards, add new cards, edit existing ones, and delete cards they no longer want. It helped me practise React Native concepts like components, props, state, and list rendering.

Overview

This project started as a simple Pokémon viewer and slowly grew into a mini CRUD app. Users can see Pokémon cards with names and numbers, and I added features to add new cards, edit existing ones, and remove cards from the list.

The focus of the app is not on heavy design, but on writing clean React Native code: breaking the UI into reusable components and managing shared state correctly.

Project Details

  • Role: Developer
  • Type: Practice project for React Native fundamentals
  • Tech Stack: React Native, JavaScript
  • Core Concepts: Components, props, state, list rendering, event handlers

Key Features

Small app, but a lot of React Native concepts inside.

Card Listing

Renders a list of Pokémon cards using map(). Each card shows a name, number, and image. This helped me practise displaying dynamic data from an array.

Add Pokémon

A controlled form lets the user type a new Pokémon’s name and number. On submit, the app uses the previous state pattern to add a new card to the list.

Edit & Delete

Each card has edit and delete actions. Editing updates a specific item in the array, while delete filters it out. This trained me to think in immutable updates.

What I Learned

This project was very important for building my React Native foundation. I became more comfortable with:

  • Managing component state and lifting state up when needed.
  • Passing data and functions between parent and child components using props.
  • Working with arrays in React Native (map, filter) for rendering and updating lists.
  • Keeping UI and data in sync when the user interacts with the app.

Next Steps
